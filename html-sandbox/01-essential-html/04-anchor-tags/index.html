HTML Basics

    Now URL stands for Uniform Resource Locator and it includes protocols
    And that response can include an HTML file. And that file is then rendered by the browser. you see the website. That HTML can include links to CSS files, JavaScript files, images and other assets.
    HTTP is the communication language for client and server
    Now, HTTP is stateless, which means that each request response cycle is independent from previous transactions.
    ex: When you refresh a web page, it's a completely different request than the last refresh and nothing carries over from the past request.
    Cookies/Session helps to maintain some level of state for http
    GET=> When you visit a website in a browser, it sends a get request to the server
    POST=> when you're sending some data, typically through a web form.
    PUT=> update, DELETE=>delete
    HTML (Hypertext Markup Language)=> Content & Structure
    It's the standard markup language for documents that are designed to be displayed in web browsers.
    HTML describes the structure of a web page semantically. HTML  Elements tell the browser how to display the content
    JavaScript=> Behavior of the website
    JavaScript is a programming language, and it can be used to make your web pages dynamic and interactive.
    Buttons/Pop ups/ sliders (JS+CSS)
    CSS (Cascading Style Sheets)
    It's a style sheet language used for describing the presentation of documents written in HTML.

Encapsulation (Access Modifiers)


Definition: Bundling data (variables) and methods (functions) that operate on the data into a single unit (class), while restricting access to some of the object's components.
Goal: Hide internal implementation details and expose only what is necessary (via public methods).

Inheritance


Definition: Inheritance allows a child class to inherit properties and behaviors from a parent class, promoting code reusability.
Goal: Reduces code duplication and improves maintainability.

Abstraction


Definition: Hiding complex implementation details and showing only the essential features of an object.
Goal: Simplify the interface and make code more user-friendly.

Polymorphism


Definition: Polymorphism allows one interface to have multiple implementations, enabling flexibility and dynamic method invocation.
Types:
Method Overriding (Runtime Polymorphism) – Child class provides a specific implementation of a method from the parent class.
Method Overloading (Compile-time Polymorphism) – Multiple methods with the same name but different parameters.
